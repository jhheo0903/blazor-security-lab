# B9. 성능 최적화 포인트

## 먼저 측정

최적화는 체감이 아니라 계측 기준으로 수행합니다.

- 렌더링 빈도
- 네트워크 호출 횟수
- 큰 목록 렌더 비용

## 주요 전략

- 불필요한 상태 변경 줄이기
- 큰 목록은 가상화 도입 검토
- 비동기 작업 병렬화/지연 로딩 적용
- 무거운 계산은 서비스로 이동

## 렌더링 관점 체크

- 같은 입력에서 매번 새 객체 생성으로 불필요 렌더 유발 여부
- 이벤트 핸들러에서 과도한 동기 블로킹 여부
- 초기 로딩에서 필요한 데이터만 먼저 가져오는지 여부

## 실무 팁

- 성능 이슈는 페이지 단위가 아니라 사용자 시나리오 단위로 분석합니다.
- 한 번에 여러 최적화를 적용하지 말고 단계적으로 검증합니다.
