# B10. 코딩 스타일 가이드

## 목표

읽기 쉬운 UI 코드와 유지보수 가능한 구조를 유지합니다.

## 규칙

- 컴포넌트는 화면 책임에 집중
- 서비스는 도메인 책임에 집중
- 모델은 의미 있는 이름과 필드만 유지
- 상태 변경 코드는 의도가 드러나게 작성
- 이벤트 핸들러 이름은 동사 기반으로 명확히 작성

## 예시 네이밍

- LoadPoliciesAsync
- ApplyFilter
- BuildDiagramNodes
- AnalyzeConflictsAsync

## 파일 구조 팁

- Pages: 기능 단위 화면
- Shared: 재사용 컴포넌트
- Services: 외부 연동/도메인 로직
- Models: 입출력/도메인 모델

## 리뷰 체크포인트

- 컴포넌트 길이가 과도하지 않은가?
- 동일 책임의 코드가 여러 파일에 분산되지 않았는가?
- 테스트 가능한 경계(서비스 인터페이스)가 존재하는가?
