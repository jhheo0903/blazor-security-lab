# B1. Blazor 기본 개념

## 무엇이 다른가

Blazor는 C#으로 UI를 작성하는 컴포넌트 프레임워크입니다.

- 화면을 컴포넌트 단위로 쪼갭니다.
- 상태 변경이 렌더링을 유도합니다.
- 이벤트/바인딩/DI를 일관된 방식으로 사용합니다.

## 최소 예시

```razor
<h3>Hello, @name</h3>
<input @bind="name" />

@code {
    private string name = "Blazor";
}
```

## C# 개발자 관점 핵심

- 컨트롤러 중심 사고보다 "상태 중심 UI" 사고가 중요합니다.
- DTO/서비스 분리는 기존 C# 백엔드 습관을 그대로 살릴 수 있습니다.
- 테스트는 UI 단위보다 서비스 단위를 먼저 안정화하는 전략이 효과적입니다.
